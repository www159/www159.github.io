Prosemirror解析-三-表情插件

发表于2021-11-10|更新于2021-11-13|源码ProseMirror
|阅读量:

本文将讲述在prosemirror中构建表情插件的过程。

NodeView

prosemirror内置了几个基本节点，他们分别是：

  • doc
  • paragraph
  • text

这三个基本节点构成了最朴素的文章样式。doc囊括全文，是文章树的根，paragraph是文章的基本分行元素，每一个行由一个paragraph组成(注:如果屏幕无限宽，那么每一段确实是一行), text则是文章的最小可操作单位，就像原子或者叶子节点。
如果我们需要创建一个节点，这个节点的内容不可以直接编辑，而是需要选中后通过类似prompt的形式来更新内容，那么NodeView就派上了用场。

需求分析

先分析一下表情节点。

  • 渲染: Unicode自带表情，基本是个浏览器就能看。
  • 输入：手动输入属实折磨。当前有几个备选方案:
    • 划框模式(以后的图片节点需要尺寸控制TOT)
    • 模糊搜索(类似于百度)
      • 基于alt
      • 基于AI
    • y滚动prompt(貌似typera就是这个)

最终我选择了第一个。考虑到该界面的动画和交互性，以及最近猛肝的react，让我有了在pm中插入react的想法。

React

考虑到表情插件的交互性以及嵌套性，我选择使用react来渲染。
这就需要解决两个问题：

数据流

  • 生命周期： react和pm都是单向数据流，因此react组件的生命周期就只有一小会——被inputrules或菜单召唤——输入挑选表情——销毁组件。这两个框架的数据流大致可以衔接。
  • 通信和事件分流：其中有一个难点——我希望实现上下左右快速选择， ctrl左右快速翻页，而这个功能需要让两个东西同时存在并共享状态。这就难办了。考虑到state修改和反复渲染的问题，我需要将sheet元素变成全局变量。

考虑到arrow事件在editor的生命周期，而nodeview的生命周期并不知道，因此需要通过editor完成plugin之间的事件交互

两个交互

prosemirror的插件具有setMeta和getMeta函数，其功能很简单。pm事件的最小周期为相邻两次update的时间差。因此在每一次update后时候分发setMeta的信息给相应对象并强制update，这样订阅者就能在下一个update前接收到信息。
但是emojiView需要在pm环境外通信，因此使用简单的eventemitter即可。

光标和顺序

还有一个痛点就是光标问题。首先鼠标点击是editor的内置事件。因此在我点完表情之后, editorView就会执行下列步骤:

  1. deselectNode
  2. setSelection

如果用同步回调,并按回车键,deselectNode后再setSelection并无作用(按回车)

因此机智的我想到了一下解决方案:

  1. 监听setSelection.
  2. 充分利用异步栈.

鼠标事件经过如下几个流程:

  1. view捕捉到mousedown并试图接管emojiView中的光标系统
  2. 接管请求被stopEvent取消,因此只能在editorView中操纵光标.
  3. view选中emoji节点.
  4. react捕捉到mouseup,并尝试将选中的表情信息发送给emojiView.
  5. emijiView调用exitView,将emoji标签的内容刷新为表情的unicode,并强制卸载react插件.
  6. arrow的事件权返回给editorView

修改逻辑

可以发现在观察栈图之前我的思路简直是一团乱麻,如果注意事件循环,完全没有这么复杂.
接下来我又尝试了如下方法.

  1. 打开emojiBar时记录在emojiState中记录openedEmojiView
  2. escapeBar捕捉到键盘事件,如果为上下左右,更新emojiBar的视图.
  3. 如果为回车, 关闭escapeBar.此时需要将close信号传给editorView.

那么问题又来了,回车和close这类往返消息不能在一次apply发生,因为事件代理是Meta

因此我又尝试给emitter构建往返消息函数invoke, 思路如下

  1. 发送回车信号,invoke一个关闭信号.
  2. react收到回车信号, 发送select index by enter信号, invoke一个关闭信号.
  3. emojiView收到select index by enter信号,在删除完毕之后返回关闭信号enter down.
  4. 设置openedEmojiView为null

有人可能会问我为什么不收到回车信号时直接设openedEmojiView为null?
嗯,我是SB.但不完全是.
因为这样写实在不太优雅.所以最终选择了异步回调.
